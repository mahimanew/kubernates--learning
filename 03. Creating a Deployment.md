
## **Step 1: Creating a Deployment**  
A **Deployment** helps manage multiple replicas of a Pod and allows rolling updates.  

1Ô∏è‚É£ Create a file named `nginx-deployment.yaml`:  
```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: nginx-deployment
  labels:
    app: nginx
spec:
  replicas: 3  # Runs 3 pods
  selector:
    matchLabels:
      app: nginx
  template:
    metadata:
      labels:
        app: nginx
    spec:
      containers:
      - name: nginx-container
        image: nginx:latest
        ports:
        - containerPort: 80
```

2Ô∏è‚É£ Apply the deployment:  
```sh
kubectl apply -f nginx-deployment.yaml
```

3Ô∏è‚É£ Verify if the Pods are running:  
```sh
kubectl get pods
```
Expected output:
```

nginx-deployment-abcdefg-1         1/1     Running   0          10s
nginx-deployment-abcdefg-2         1/1     Running   0          10s
nginx-deployment-abcdefg-3         1/1     Running   0          10s
```

4Ô∏è‚É£ Check the Deployment:  
```sh

```

## **Step 2: Exposing the Deployment with a Service**  
Since our **nginx deployment** is running, we now expose it externally using a **LoadBalancer Service**.

1Ô∏è‚É£ Expose the deployment:  
```sh

```

2Ô∏è‚É£ Check the Service:  
```sh

```
Expected output:
```

nginx-deployment    LoadBalancer   10.96.0.2       <pending>     80:XXXXX/TCP   5s
```
üìå If using **Minikube**, run the following to get the external URL:  

üìå If using **Kind**, port-forward the service to access it:

## **Step 4: Rolling Updates & Rollbacks**  
Let‚Äôs update the **nginx** version in the deployment.  

1Ô∏è‚É£ Edit the deployment file:  
```yaml
containers:
  - name: nginx-container
    image: nginx:1.19  # Update version
```

2Ô∏è‚É£ Apply the update:  
```sh
kubectl apply -f nginx-deployment.yaml
```

3Ô∏è‚É£ Check the rollout status:  
```sh
kubectl rollout status deployment/nginx-deployment
```

4Ô∏è‚É£ If something goes wrong, rollback to the previous version:  
```sh
kubectl rollout undo deployment/nginx-deployment
```
